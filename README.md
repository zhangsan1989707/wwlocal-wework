# 政务微信开放数据审计平台

## 项目简介

政务微信数据查询平台 -- 从政务微信开放接口获取加密业务日志，RSA+AES 双层解密后存储到 MySQL，前端提供查询/同步/密钥管理功能。

## 功能特性

- **日志审计查询**：多类型日志解密查询，支持条件筛选、手机号匹配、实时查询、CSV 导出
- **数据同步**：全量同步（按时间范围）、增量同步（断点续拉）、定时自动同步
- **通讯录管理**：组织架构树、成员查询、全量/增量同步
- **密钥管理**：多版本 RSA 密钥、在线切换、密钥测试
- **数据类型配置**：前端启用/禁用同步数据类型
- **总览看板**：KPI 概览、同步状态、异常告警、部门活跃度分析
- **操作审计**：自动记录所有 API 操作
- **系统状态**：DB 连通性、密钥状态、同步覆盖率、存储空间

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Go 1.21 + Echo v4 + GORM + MySQL 8.0 |
| 前端 | Vue 3.5 + TypeScript + Element Plus + Vite 8 |
| 容器 | Docker + docker-compose |
| 加密 | RSA PKCS1v15 + AES-128-CBC |

## 快速开始

### 开发环境

```bash
git clone <repo-url>
cd wwlocal-wework
cp .env.example .env
# 编辑 .env 填入实际配置
docker-compose up -d --build
```

前端开发服务器默认运行在 `http://localhost:5173`，后端 API 在 `http://localhost:3010`。

参见 [部署指南](docs/部署指南.md)。

## 目录结构

```
wwlocal-wework/
├── cmd/server/              # 入口，组合根
├── config/                  # 配置加载，DB_* 环境变量覆盖
├── internal/
│   ├── handler/             # HTTP 处理层
│   ├── service/             # 业务逻辑层
│   ├── repository/          # 数据访问层
│   ├── model/               # 数据模型
│   ├── middleware/          # 中间件 (认证、限流)
│   ├── router/              # Echo 路由注册
│   └── crypto/              # RSA/AES 解密实现
├── pkg/
│   ├── response/            # 统一 JSON 响应封装
│   └── httpclient/          # 通用 HTTP 客户端
├── web/                     # Vue 前端项目
├── deploy/                  # 离线部署包
├── docs/                    # 项目文档
├── keys/                    # RSA 私钥文件 (各版本目录)
├── docker-compose.yml
├── config.yaml
└── .env                     # 环境变量 (不提交)
```

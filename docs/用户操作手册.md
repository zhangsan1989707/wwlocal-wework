# 政务微信数据审计平台 - 用户操作手册

## 1. 登录

**访问地址**：打开浏览器，访问 `http://服务器IP:18073`

**登录方式**：
- 输入管理员用户名和密码
- 点击"登录"按钮
- 系统使用 JWT 进行身份认证，令牌有效期 **24 小时**
- 令牌过期后，系统自动清除本地登录状态并跳转回登录页

**安全机制**：
- 密码使用 bcrypt 哈希存储，不可逆
- 同一 IP 地址 1 分钟内连续 5 次登录失败将被临时锁定，1 分钟后自动解锁
- 登录成功后，令牌存储在浏览器 localStorage 中，关闭浏览器后仍有效，直到过期或手动退出
- 超级管理员可查看全部数据；部门管理员仅能查看授权部门及其子部门成员相关数据

**退出**：点击页面右上角"退出"按钮，清除本地令牌并返回登录页。

## 2. 总览看板

登录后默认进入总览看板页面，展示系统核心运行指标。

### 2.1 KPI 卡片

页面顶部展示 6 个关键指标卡片：

| 卡片名称 | 含义说明 |
|---------|---------|
| 最近同步时间 | 上一次数据同步完成的时间点 |
| 7 天同步记录数 | 最近 7 天内成功同步的日志记录总数 |
| 失败类型数 | 同步历史中出现异常的同步类型数量 |
| 活跃密钥版本 | 当前用于解密数据的 RSA 密钥版本号 |
| 通讯录人数 | 当前通讯录中已同步的成员总数 |
| 未使用人员比例 | 超过指定天数未产生任何日志的人员占总人数的比例 |

### 2.2 异常告警

看板会自动检测以下异常情况并以告警条形式展示：

| 告警类型 | 触发条件 | 告警级别 |
|---------|---------|---------|
| 同步失败 | 存在同步失败记录 | 红色（严重） |
| 密钥超期 | 当前活跃密钥使用超过 90 天未更换 | 黄色（警告） |
| 通讯录过期 | 通讯录超过 7 天未执行同步 | 黄色（警告） |
| 无新数据 | 最近 24 小时内没有任何新同步的日志数据 | 黄色（警告） |

### 2.3 使用率分析

看板提供通讯录成员使用率分析功能：

**筛选条件**：
- **时间范围**：选择分析的起止时间
- **部门**：选择特定部门进行分析，或选择"全部部门"
- **最低未使用天数**：设置阈值，只显示超过该天数未产生日志的人员

**分析结果**：
- **部门统计表**：展示各部门成员数量、未使用人数、未使用比例
- **未使用人员列表**：列出符合条件的具体人员信息

**操作功能**：
- 点击人员姓名可跳转到日志查询页面，自动填入该人员的查询条件
- 点击"导出 CSV"按钮将当前未使用人员列表导出为 CSV 文件

## 3. 日志审计

用于查询和检索已同步的业务日志数据。

### 3.1 查询条件

| 条件项 | 说明 |
|-------|------|
| 日志类型 | 下拉多选框，可选择一个或多个数据类型进行过滤 |
| 时间范围 | 起止时间选择器，提供快捷按钮（今天/昨天/近7天/近30天/自定义） |
| 成员标识/手机号 | 输入手机号或 UserID；手机号会自动解析为通讯录 UserID 后匹配日志中 `openid/userid` 相关字段 |
| 实时查询 | 开关控制。开启时直接从政务微信接口实时拉取数据；关闭时仅查询本地已同步数据 |
| 自定义筛选条件 | 支持添加多组条件，每组包含"字段名 + 操作符 + 值" |

**自定义筛选条件**：
- **字段名**：输入 JSON 路径（如 `user_id`、`content.msg`），支持自动补全
- **操作符**：支持 `=`（精确匹配）和 `like`（模糊匹配，使用 `%` 通配符）
- **值**：输入匹配内容
- 点击"+"可添加多组条件，多组条件之间为 AND 关系

### 3.2 查询结果

- **分页展示**：查询结果按时间倒序排列，支持分页浏览
- **展开详情**：点击某条记录可展开查看完整的 JSON 原始数据
- **导出 CSV**：点击"导出"按钮，将当前查询条件对应的数据导出为 CSV 文件
- **字段路径**：系统自动从已有数据中提取可用的 JSON 字段路径，用于自定义筛选条件的自动补全

### 3.3 行为查询

用于按单个成员查看跨日志类型的行为时间线。

**入口**：
- 顶部菜单进入"行为查询"
- 通讯录成员详情中点击"查看行为轨迹"，系统会自动带入该成员手机号并发起查询

**查询条件**：
- **OpenID 或手机号**：输入成员 OpenID、手机号或 UserID
- **时间范围**：支持快捷选择，单次查询最大 31 天
- **日志类型**：可勾选指定类型；不勾选时按系统启用的数据类型查询

**结果说明**：
- 时间线按日志时间倒序展示
- 每条记录显示日志类型、命中字段、摘要和原始结构化数据
- 系统会尽量把命中的手机号/UserID 补充显示为通讯录姓名
- 微盘 `90000061/90000062/90000063` 支持按 `oper_name` 查询
- `90000066` API 调用日志暂不支持行为查询，结果摘要中会显示跳过原因

**导出 CSV**：
- 点击"导出"按钮，可导出当前条件下的行为时间线 CSV
- 导出列包括：时间、日志类型编号、日志类型名称、命中字段、摘要、数据详情

**权限限制**：
- 超级管理员可查询全部成员
- 部门管理员只能查询授权部门及其子部门成员
- 部门管理员使用通用日志查询时必须指定本部门成员手机号或 UserID，不能进行无成员范围的全局日志搜索

## 4. 同步任务

管理从政务微信接口拉取数据的同步任务。

### 4.1 定期同步（自动调度）

系统可按配置的时间间隔自动执行增量同步。

**操作**：
- **启动**：点击"启动"按钮，系统按设定间隔定期拉取新数据
- **停止**：点击"停止"按钮，暂停自动调度
- **修改间隔**：在下拉框中选择延迟时间（10 分钟 / 30 分钟 / 1 小时），点击"更新"生效

**状态展示**：
- 当前调度状态（运行中 / 已停止）
- 同步间隔
- 上次执行时间
- 下次执行时间

### 4.2 全量同步

手动触发一次完整的数据拉取。

**操作步骤**：
1. 选择时间范围：使用时间选择器设定拉取数据的起止时间，提供快捷按钮（今天 / 近7天 / 近30天 / 自定义）
2. 选择数据类型：勾选需要同步的日志类型（可多选或全选）
3. 点击"开始同步"按钮，系统在后台异步执行
4. 同步过程中可点击"取消"按钮中断当前任务

### 4.3 立即增量同步

点击"立即执行"按钮，系统从上次同步的断点（`sync_state.last_log_time`）继续拉取新数据。无需配置时间范围和数据类型，自动同步所有启用的类型。

### 4.4 同步状态

同步执行过程中，页面实时展示以下信息：
- **进度条**：显示整体同步进度百分比
- **当前数据类型**：正在同步的 feature 名称
- **结果表格**：按数据类型展示同步结果（成功数 / 失败数 / 跳过数）

### 4.5 同步历史

页面底部展示历史同步记录表格：

| 列名 | 说明 |
|-----|------|
| 执行时间 | 同步任务的开始时间 |
| 数据类型 | 同步的日志类型 |
| 触发方式 | 手动触发 / 定时调度 / 增量同步 |
| 成功数 | 成功入库的记录数 |
| 失败数 | 失败的记录数 |
| 耗时 | 本次同步的总耗时 |

## 5. 通讯录

管理从政务微信获取的组织架构和人员信息。

### 5.1 同步操作

页面顶部提供三个操作按钮：

- **全量同步**：拉取所有部门和成员信息，耗时较长，适用于首次同步或数据异常时重建
- **增量同步**：仅拉取新增或变更的成员信息，速度较快
- **取消**：中断正在进行的同步任务

同步状态通过进度条实时展示。

### 5.2 组织架构树

页面左侧展示组织架构树形结构：
- 点击某个部门节点，右侧成员列表自动过滤为该部门的成员
- 顶部搜索框支持按部门名称快速定位

### 5.3 成员列表

- 展示成员姓名、手机号、部门、职位等信息
- 支持按姓名或手机号搜索过滤
- 点击某条成员记录可查看详细信息

### 5.4 详情抽屉

点击成员后，右侧弹出详情抽屉面板，展示：
- 姓名、手机号、邮箱、部门、职位等个人基本信息
- "查询日志"按钮：点击跳转到日志查询页面，自动填入该成员的手机号作为查询条件
- "查看行为轨迹"按钮：点击跳转到行为查询页面，自动填入该成员的手机号并查询行为时间线

## 6. 密钥管理

管理用于解密业务数据的 RSA 密钥。

### 6.1 密钥列表

页面展示所有已注册的 RSA 密钥版本：

| 列名 | 说明 |
|-----|------|
| 版本号 | 密钥的唯一标识版本号 |
| 状态 | 活跃（当前使用中）/ 备用（历史版本，用于解密失败时回退） |
| 创建时间 | 密钥添加到系统的时间 |

### 6.2 切换活跃密钥

- 点击某个备用密钥的"设为活跃"按钮
- 确认后，后续解密操作将优先使用新密钥
- 原活跃密钥自动降级为备用状态
- 此操作仅影响后续新数据的解密，已解密的历史数据不受影响

### 6.3 测试密钥

- 点击"测试"按钮，系统验证密钥格式是否正确
- 测试结果展示密钥的位数（如 2048 位）和类型（如 RSA Private Key）

### 6.4 添加新密钥

1. 点击"添加密钥"按钮
2. 输入版本号（如 `v2`，需唯一）
3. 粘贴 PEM 格式的 RSA 私钥内容（`-----BEGIN RSA PRIVATE KEY-----` 开头）
4. 点击"确定"保存

密钥文件存储在服务器的 `keys/{version}/rsa_private_key.pem`，系统启动时会自动检查权限，若权限过于宽松会修正为 `600`。

## 7. 数据类型

管理需要同步的日志数据类型（feature）。

### 7.1 启用/禁用

- 页面以表格形式列出所有已配置的数据类型
- 每行提供"启用/禁用"开关，控制该类型是否参与同步
- 禁用后，定时调度和全量同步将跳过该类型

### 7.2 批量操作

- 提供"全选"复选框，勾选后可批量启用或禁用
- 批量操作按钮："批量启用"、"批量禁用"

### 7.3 同步状态

表格展示各类型的同步状态：
- 最近同步时间
- 累计同步记录数
- 最后一次同步结果（成功 / 失败）

### 7.4 查看最近日志

每行提供"查看最近日志"链接，点击后跳转到日志查询页面，自动选中该数据类型并展示最新的记录。

## 8. 操作审计

查看系统操作日志，用于审计用户行为。

### 8.1 筛选条件

| 条件 | 说明 |
|-----|------|
| 操作类型 | 下拉选择，可选值：登录（login）、同步（sync）、查询（query）、密钥管理（key_management）、调度（scheduler）、通讯录（contacts）等 |
| 状态 | 全部 / 成功 / 失败 |

### 8.2 日志表格

| 列名 | 说明 |
|-----|------|
| 操作时间 | 请求发生的时间 |
| 操作用户 | 执行操作的用户名（未登录操作显示为空） |
| 操作类型 | 对应的操作分类 |
| 请求方法 | HTTP 方法（GET / POST / PUT） |
| 请求路径 | API 接口路径 |
| 状态码 | HTTP 响应状态码 |
| 耗时 | 请求处理耗时（毫秒） |

### 8.3 说明

- 操作审计中间件对所有请求自动记录（`/health` 排除）
- 登录接口的记录中，未登录时 username 为空，登录成功后 username 为登录用户名
- 日志记录在后台异步写入，不影响请求响应速度

---

## 9. 系统状态

查看系统运行状态和存储使用情况。

### 9.1 基础状态

| 指标 | 说明 |
|-----|------|
| 数据库连通性 | 与 MySQL 的连接是否正常 |
| 系统运行时间 | 从上次启动至今的持续时间 |
| 密钥状态 | 当前活跃密钥版本及密钥总数 |
| 通讯录状态 | 通讯录最近同步时间及成员总数 |

### 9.2 同步覆盖率

展示各数据类型的同步覆盖率：
- 类型名称
- 最近同步时间
- 累计记录数
- 覆盖的时间范围

### 9.3 数据库存储空间

展示各日志表（`log_{feature_id}_{YYYYMM}`）的存储占用：
- 表名
- 记录数
- 数据大小
- 索引大小
- 总占用空间

### 9.4 注意事项

- 系统状态数据通过后台定时采集，展示为快照信息
- 若数据库连接异常，页面会以红色警告提示
- 动态日志表按 feature ID 和月份自动创建，无需手动管理

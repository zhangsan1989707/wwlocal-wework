# 部署指南

## 环境要求

| 项目 | 要求 |
|------|------|
| Docker | >= 20.10 |
| docker-compose | >= 1.29 |
| 内存 | >= 4 GB |
| 磁盘 | >= 20 GB |

## 环境变量配置 (.env)

从 `.env.example` 复制并编辑：

```bash
cp .env.example .env
```

| 变量 | 说明 | 是否必填 | 示例 |
|------|------|----------|------|
| `DB_PASSWORD` | MySQL root 密码 | **必填** | `StrongPass123!` |
| `WEWORK_BASE_URL` | 政务微信 API 地址 | **必填** | `https://10.1.214.27:89` |
| `WEWORK_CORPID` | 企业 ID | **必填** | `wwe8a4b3c2d1e0f` |
| `WEWORK_SECRET` | 日志应用 Secret | **必填** | `your_secret` |
| `WEWORK_CONTACT_SECRET` | 通讯录应用 Secret | **必填** | `your_contact_secret` |
| `WEWORK_INSECURE_SKIP_VERIFY` | 跳过 TLS 证书验证 | 选填 | `false`（自签证书设为 `true`） |
| `AUTH_USERNAME` | 登录用户名 | 选填 | `admin`（默认 `admin`） |
| `AUTH_PASSWORD` | 登录密码 | **必填** | `Admin@123456` |
| `JWT_SECRET` | JWT 签名密钥 | **必填** | 随机长字符串 |
| `KEY_ENCRYPT_KEY` | 密钥加密 Key（32 字节 HEX） | 选填 | `a1b2c3...` |

> **安全提示**：`JWT_SECRET` 和 `KEY_ENCRYPT_KEY` 应使用足够强度的随机值，切勿提交到版本控制。

## 开发环境部署

1. **克隆项目**

```bash
git clone <repo-url>
cd wwlocal-wework
```

2. **配置环境变量**

```bash
cp .env.example .env
# 编辑 .env，填入实际值
```

3. **启动服务**

```bash
docker-compose up -d --build
```

4. **验证服务**

```bash
# 后端健康检查
curl http://localhost:3010/health

# 前端访问
open http://localhost:5173
```

## 生产环境部署

使用项目自带的离线部署包进行部署。

1. **上传部署包**

将 `deploy/` 目录及 `docker-compose.yml`、`config.yaml`、`config/` 等文件打包上传到服务器。

2. **配置环境变量**

```bash
cp .env.example .env
# 编辑 .env，填入实际值（必填项全部填写）
```

3. **执行部署脚本**

```bash
cd deploy
chmod +x deploy.sh
./deploy.sh
```

4. **验证服务**

```bash
curl http://localhost:3010/health
# 应返回 {"status":"ok"}
```

## 端口映射

| 服务 | 容器内端口 | 宿主映射端口 | 修改方式 |
|------|-----------|-------------|---------|
| MySQL | 3306 | **3307** | `docker-compose.yml` mysql ports |
| 后端 API | 8080 | **3010** | `docker-compose.yml` backend ports |
| 前端 (dev) | 5173 | **5173** | `web/vite.config.ts` server.port |

修改端口后需同步更新前端代理配置（`web/vite.config.ts` 中 `proxy` 指向的后端地址）。

## MySQL 调优

项目在 `docker-compose.yml` 中挂载了 `mysql/my.cnf`，关键参数说明：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| `innodb_buffer_pool_size` | 内存的 70-80% | InnoDB 缓冲池大小，核心性能参数 |
| `innodb_log_file_size` | `256M` | Redo log 文件大小 |
| `innodb_flush_log_at_trx_commit` | `1` | 事务安全级别（`1`=每次提交落盘） |
| `max_connections` | `200` | 最大并发连接数 |
| `character-set-server` | `utf8mb4` | 默认字符集 |

生产环境建议根据服务器内存调整 `innodb_buffer_pool_size`（例如 8G 内存可设为 `6G`）。

## 健康检查

```bash
curl http://localhost:3010/health
```

期望响应：

```json
{
  "status": "ok"
}
```

如需持续监控，可使用以下命令定时检查：

```bash
watch -n 30 'curl -sf http://localhost:3010/health || echo "UNHEALTHY"'
```

## 常见问题

### 无法连接政务微信

- 检查 `WEWORK_BASE_URL` 是否正确（包含协议和端口）
- 如使用自签证书，将 `WEWORK_INSECURE_SKIP_VERIFY` 设为 `true`
- 确认服务器网络可访问政务微信 API 地址
- 确认 `WEWORK_CORPID` 和 `WEWORK_SECRET` 填写正确

### MySQL 连接失败

- 检查 `DB_PASSWORD` 是否与 `docker-compose.yml` 中 MySQL 配置一致
- 确认 MySQL 容器已正常启动：`docker-compose ps`
- 查看 MySQL 日志：`docker-compose logs mysql`

### 前端无法访问

- 确认前端服务已启动：`docker-compose ps`
- 检查端口 5173 是否被占用：`lsof -i :5173`
- 开发环境下确认已执行 `cd web && npm install && npm run dev`
- 生产环境前端通过 Nginx 提供服务，检查 Nginx 容器状态

### 解密失败

- 确认 RSA 私钥已正确上传（通过密钥管理页面）
- 确认私钥版本已激活
- 检查 `keys/` 目录下的 PEM 文件权限

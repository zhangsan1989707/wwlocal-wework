### 1、创建群聊

创建一个包含应用的群聊会话。

**请求方式：** POST

**请求地址：** https://uat.rztcd.cn/cgi-bin/appchat/create?access_token=ACCESS_TOKEN

**请求包体：**

```
{    "name": "应用群聊",    "owner": "user1",    "userlist": [        "user1",        "user2"    ],    "chatid": ""}
```

**参数说明：**

| 参数名称 | 是否必须 | 说明                                                         |
| -------- | -------- | ------------------------------------------------------------ |
| name     | 否       | 群聊名，最多50个utf8字符，超过将截断                         |
| owner    | 否       | 指定群主的id。如果不指定，系统会随机从userlist中选一人作为群主 |
| userlist | 是       | 群成员id列表。至少2人，至多2000人（如果指定了owner，则owner也需要在userlist中） |
| chatid   | 否       | 群聊的唯一标志，不能与已有的群重复；字符串类型，最长32个字符。只允许字符0-9及字母a-zA-Z。如果不填，系统会随机生成群id |

**返回结果：**

```
{    "errcode": 0,    "errmsg": "ok",    "chatid": "Chat_Id",}
```

**返回结果说明：**

| 参数名称 | 说明                       |
| -------- | -------------------------- |
| errcode  | 0：正常返回 非0异常        |
| errmsg   | 错误提示信息               |
| chatid   | 系统生成的唯一标志群聊的ID |

**限制说明：**

1. 只能操作在当前应用可见范围的成员；
2. 只有当前应用创建的群聊，当前应用可以操作；
3. 群成员人数不可超过管理端配置的“群成员人数上限”（通用的上限），且最大不可超过2000人；

> 注意：刚创建的群，如果没有下发消息，在客户端不会出现该群。

### 2、修改群聊

修改已有的群聊会话。

**请求方式：** POST

**请求地址：** https://uat.rztcd.cn/cgi-bin/appchat/update?access_token=ACCESS_TOKEN

**请求包体：**

```
{    "chatid": "Chat_Id",    "name": "Name",    "owner": "user1",    "add_user_list": ["user2", "user3"],    "del_user_list": ["user4", "user5"],}
```

**参数说明：**

| 参数名称      | 是否必须 | 说明                                                         |
| ------------- | -------- | ------------------------------------------------------------ |
| chatid        | 是       | 要修改的群聊ID                                               |
| name          | 否       | 新的群聊名。若不需更新，请忽略此参数。最多50个utf8字符，超过将截断 |
| owner         | 否       | 新群主的id。若不需更新，请忽略此参数                         |
| add_user_list | 否       | 要添加成员的id列表                                           |
| del_user_list | 否       | 要移出成员的id列表                                           |

**返回结果：**

```
{    "errcode": 0,    "errmsg": "ok"}
```

**返回结果说明：**

| 参数名称 | 说明                |
| -------- | ------------------- |
| errcode  | 0：正常返回 非0异常 |
| errmsg   | 错误提示信息        |

**限制说明：**

1. 只能操作在当前应用可见范围的成员；
2. 只有当前应用创建的群聊，当前应用可以操作；
3. 群成员人数不可超过管理端配置的“群成员人数上限”（通用的上限），且最大不可超过2000人；

### 3、获取群聊会话

获取已有的群聊会话信息。

**请求方式：** GET

**请求地址：** https://uat.rztcd.cn/cgi-bin/appchat/get?access_token=ACCESS_TOKEN&chatid=CHAT_Id

**参数说明：**

| 参数名称 | 是否必须 | 说明           |
| -------- | -------- | -------------- |
| chatid   | 是       | 要查询的群聊ID |

**返回结果：**

```
{    "errcode": 0,    "errmsg": "ok",    "chat_info": {        "chatid": "Chat_Id",        "name": "Name",        "owner": "user1",        "userlist": [user1, user2]    },    "custom_user_list": [{        "userid": "user1",        "nickname": "user1name",        "order": 0,    },{        "userid": "user2",        "nickname": "user2name",        "order": 1,    }]}
```

**返回结果说明：**

| 参数名称         | 说明                |
| ---------------- | ------------------- |
| errcode          | 0：正常返回 非0异常 |
| errmsg           | 错误提示信息        |
| chat_info        | 群聊信息            |
| chatid           | 群聊ID              |
| name             | 群聊名称            |
| owner            | 群主ID              |
| userlist         | 群成员id列表        |
| custom_user_list | 群成员详情          |

### 5、推送应用消息

以应用身份向群聊会话中推送消息，支持文本、图片、视频、文件、图文等类型。

**请求方式：** POST

**请求地址：** https://uat.rztcd.cn/cgi-bin/appchat/send?access_token=ACCESS_TOKEN

**请求包体：**

```
{    "chatid": "Chat_Id",    "msgtype": "",    "text": {},    "image": {},    "voice": {},    "video": {},    "file": {},    "textcard": {},    "news": {},    "mpnews": {},    "markdown": {}}
```

**参数说明：**

| 参数名称 | 是否必须 | 说明                                                         |
| -------- | -------- | ------------------------------------------------------------ |
| chatid   | 是       | 要发送消息的群聊ID                                           |
| msgtype  | 是       | 要发送的消息类型，可选类型有text、image、voice、video、file、textcard、news和mpnews几种 |
| text     | 否       | 当消息类型为text时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”文本消息”的text字段 |
| image    | 否       | 当消息类型为image时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”图片消息”的image字段 |
| voice    | 否       | 当消息类型为voice时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”语音消息”的voice字段 |
| video    | 否       | 当消息类型为video时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”视频消息”的video字段 |
| file     | 否       | 当消息类型为file时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”文件消息”的file字段 |
| textcard | 否       | 当消息类型为textcard时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”文本卡片消息”的textcard字段 |
| news     | 否       | 当消息类型为news时需指定该字段，内容参考API文档->”消息推送”->”主动发送消息”中”图文消息”的news字段 |
| mpnews   | 否       | 当消息类型为mpnews时需指定该字段，内容见下文定义             |
| markdown | 否       | 当消息类型为markdown时需指定该字段，内容见下文定义           |

**返回结果：**

```
{    "errcode": 0,    "errmsg": "ok",    "jobid": "4_1603337930_170948"}
```

**返回结果说明：**

| 参数名称 | 说明                                                         |
| -------- | ------------------------------------------------------------ |
| errcode  | 0：正常返回 非0异常                                          |
| errmsg   | 错误提示信息                                                 |
| jobid    | 任务id，可以通过jobid查询消息发送的进度，具体请求接口见获取异步任务结果 |

### 6、撤回应用群聊消息

撤回应用推送的群聊消息。

**请求方式：** POST

**请求地址：** https://uat.rztcd.cn/cgi-bin/appchat/revoke?access_token=ACCESS_TOKEN

**请求包体：**

```
{    "jobid":  "4_1603337930_170948"  ,    "revokelist": [{       "userid": "evan001"  ,       "msgid": ["CAASLHd3b3Blbm1zZ181NjI5NDk5NjMzMjEzMTIwXzE2MDM4NzczMjdfMTAxNzc0GM/z5PwF", ..., "CAASLHd3b3Blbm1zZ181NjI5NDk5NjMzMjEzMTIwXzE2MDM4NzczMjdfMTAxNzc0GM/z5PwF"]    }, {       "userid": "test1"  ,       "msgid": ["CAASLHd3b3Blbm1zZ181NjI5NDk5NjMzMjEzMTIwXzE2MDM4NzczMjdfMTAxNzc0GM/z5PwF", ..., "CAASLHd3b3Blbm1zZ181NjI5NDk5NjMzMjEzMTIwXzE2MDM4NzczMjdfMTAxNzc0GM/z5PwF"]    }]}
```

**参数说明：**

| 参数名称   | 是否必须 | 说明                                     |
| ---------- | -------- | ---------------------------------------- |
| jobid      | 否       | 应用推送消息的任务ID, 由推送接口返回     |
| revokelist | 否       | 成员ID列表, 需要撤回的成员ID，限制2000人 |
| msgid      | 否       | 消息ID, 由异步任务结果查询接口返回       |
| userid     | 否       | 成员ID列表, 需要撤回的成员ID             |

> 注意：jobid 与 revokelist 必须填写其中一项, 如果填写了 jobid 则会忽略 revokelist, 撤回该 jobid 涉及的所有消息

**返回结果：**

```
{    "errcode": 0,    "errmsg": "ok"}
```

**返回参数说明：**

| 参数名称 | 说明                |
| -------- | ------------------- |
| errcode  | 0：正常返回 非0异常 |
| errmsg   | 错误提示信息        |

#### 消息类型定义

1、图文消息（mpnews）

> mpnews类型的图文消息，跟普通的图文消息一致，唯一的差异是图文内容存储在本地。
> 多次发送mpnews，会被认为是不同的图文，阅读、点赞的统计会被分开计算。

**请求示例：**

```
{    "chatid": "Chat_Id",    "msgtype": "mpnews",    "mpnews": {        "articles":[            {                "title": "地球一小时",                "thumb_media_id": "1G6nrLmr5EC3MMb_-zK1dDdzmd0p7cNliYu9V5w7o8K0",                "author": "Author",                "content_source_url": "https://uat.rztcd.cn",                "content": "3月24日20:30-21:30 \n办公区将关闭照明一小时，请各部门同事相互转告",                "digest": "3月24日20:30-21:30 \n办公区将关闭照明一小时"            }         ]    }}
```

**参数说明：**

| 参数名称           | 是否必须 | 说明                                                         |
| ------------------ | -------- | ------------------------------------------------------------ |
| chatid             | 是       | 要发送消息的群聊ID                                           |
| msgtype            | 是       | 消息类型，此时固定为：mpnews                                 |
| mpnews             | 是       | 消息内容                                                     |
| articles           | 是       | 图文消息，一个图文消息支持1到8条图文                         |
| title              | 是       | 标题，不超过128个字节，超过会自动截断                        |
| thumb_media_id     | 是       | 图文消息缩略图的media_id, 可以通过素材管理接口获得。此处thumb_media_id即上传接口返回的media_id |
| author             | 否       | 图文消息的作者，不超过64个字节                               |
| content_source_url | 否       | 图文消息点击“阅读原文”之后的页面链接                         |
| content            | 是       | 图文消息的内容，支持html标签，不超过666K个字节               |
| digest             | 否       | 图文消息的描述，不超过512个字节，超过会自动截断              |

2、markdown消息

> 目前仅支持[markdown语法的子集](https://uat.rztcd.cn:89/api/doc#10167/支持的markdown语法)

**请求示例：**

```
{    "chatid": "CHATID",    "msgtype": "markdown",    "markdown": {        "content": "您的会议室已经预定，稍后会同步到`邮箱`\n>**事项详情**\n>事 项：<font color=\\\"info\\\">开会</font>\n>组织者：@miglioguan\n>参与者：@miglioguan、@kunliu、@jamdeezhou、@kanexiong、@kisonwang\n>\n>会议室：<font color=\\\"info\\\">广州TIT 1楼 301</font>\n>日 期：<font color=\\\"warning\\\">2018年5月18日</font>\n>时 间：<font color=\\\"comment\\\">上午9:00-11:00</font>\n>\n>请准时参加会议。\n>\n>如需修改会议信息，请点击：[修改会议信息](https://uat.rztcd.cn)"    }}
```

**参数说明：**

|  参数   | 是否必须 |                        说明                        |
| :-----: | :------: | :------------------------------------------------: |
| chatid  |    是    |                       群聊id                       |
| msgtype |    是    |           消息类型，此时固定为：markdown           |
| content |    是    | markdown内容，最长不超过2048个字节，必须是utf8编码 |

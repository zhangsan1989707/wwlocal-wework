### 2.日志解密

在管理端开启“数据开放服务”时，需上传公私钥对，用以加密日志数据。同时为支持公私钥对定期更换，必须记住公私钥对和其版本的对应关系。

1.**如何生成RSA公私钥**

使用openssl生成RSA公私钥对，具体如下述命令：

先生成私钥:

```
openssl genrsa -out rsa_private_key.pem 1024
```

再生成公钥：

```
openssl rsa -in rsa_private_key.pem -pubout -out rsa_public_key.pem
```

2.**解密日志**

- 第一步先使用PKCS1 padding模式，RSA私钥对`enc_key`进行解密，得到日志数据的加密`aes_key`
- 第二步，AES128算法，CBC，PKCS7模式，以`aes_key`为加密key解密`enc_data`，得到日志原文

#### 3.1 消息内容Content的格式定义

1. msg_type = 1 文本消息，包含纯文本、@消息和表情或三者组合

   ```
   { "text_msg": [{     "text_type": 1,  // 1 纯文本     "text": "abc" }, {     "text_type": 2,  // @消息     "at_msg": {         "at_type": 1, // 1 指定用户 2@所有人         "at_user": [{               "openid": "",               "type": 0          }]     } }, {     "text_type": 3,  // 表情     "emotion": {         "shortcut": "[惊讶]"     } }]}
   ```

2. msg_type = 2 文件消息

   ```
   { "file_msg": [{     "size": 100,     "name": "",     "md5": "",     "fileid": "" // 文件唯一ID }]}
   ```

3. msg_type = 3 语音消息

   ```
   { "voice_msg": {    "size": 100,    "name": "",    "md5": "",    "fileid": "", // 文件唯一ID    "duration": 10 }}
   ```

4. msg_type=4 视频消息

   ```
   { "video_msg": {     "size": 100,     "name": "",     "md5": "",     "fileid": "", // 文件唯一ID     "duration": 10 }}
   ```

5. msg_type=5 位置消息

   ```
   { "location_msg": {     "longitude": 113.955491,     "latitude": 22.540505,     "address": "",     "title": "" }}
   ```

6. msg_type=6 链接

   ```
   { "link_msg": {     "link_url": "",     "image_url": "",     "title": "",     "description": "" }}
   ```

7. msg_type=7 个人名片

   ```
   { "card_msg": {}}
   ```

   > 个人名片消息暂不支持获取消息实体。

8. msg_type=8 红包

   ```
   {"hongbao_msg": {}}
   ```

   > 红包消息暂不支持获取消息实体。

9. msg_type=9 阅后即焚消息

   ```
   {"snapshot":{}}
   ```

   > 阅后即焚消息暂不支持获取消息实体

10. msg_type=10 应用发送的图文卡片消息

    ```
    {"news_msg": {"articles": [{    "title": "",    "description": "",    "url": "",    "picurl": "",    "btn_txt": ""}],"link_type": 1 //1 图文 2 外链图文}}
    ```

11. msg_type=11 应用发送的文本卡片消息

    ```
    {"text_card": {    "title": "",    "description": "",    "url": "",    "btn_txt": ""}}
    ```

12. msg_type=12，音视频

    ```
    {"voip_msg": {}}
    ```

13. msg_type = 13 图片消息

    ```
    {"file_msg":[{   "size": 100,   "name": "",   "md5": "",   "fileid": "" // 文件唯一ID}]}
    ```

14. msg_type=14，自定义表情

    ```
    {"user_emotion_msg": {    "type": 1,  // 1 静态 2 动态    "width": 0, // 宽度    "height": 0,  // 高度    "fileid": "", // 表情文件ID    "md5": "",    "size": 0}}
    ```

15. msg_type=15，聊天记录

    ```
    {"forward_msgs": {    "title": "",    "items": [{        "send_time": 0,        "msg_type": 1,        "content": ""    }]}}
    ```

16. msg_type=16，长文本

    ```
    {"long_text_msg": [{    "text_type": 1,  // 1 纯文本    "text": "abc",    "chunk_total":11, //长文本分片数量    "chunk_num":1 //当前是第几片分片，从1开始计数}, {    "text_type": 2,  // @消息    "at_msg": {        "at_type": 1, // 1 指定用户 2@所有人        "at_user": [{            "openid": "",            "type": 0        }]    }}, {    "text_type": 3,  // 表情    "emotion": {        "shortcut": "[惊讶]"    }}]}
    ```

### 4.错误码说明

#### 4.1 消息发送失败常见错误码

| 错误码    | 含义                         |
| --------- | ---------------------------- |
| 11        | 无网络连接                   |
| 8000      | 服务异常，可能正在升级       |
| -2004     | 非法用户                     |
| -2097     | 低版本需要升级云服务         |
| -3001     | 群聊获取群id失败             |
| -3008     | 群已解散                     |
| -4011     | 发送失败，客户端需要重新发送 |
| -4014     | 非对方好友，不能发送消息     |
| -4019     | 被群主禁言                   |
| -4020     | 全员禁言                     |
| -17000101 | 临时会话已过期               |
| -17000102 | 聊天申请未通过               |
| -17000100 | 无权限发起聊天               |
